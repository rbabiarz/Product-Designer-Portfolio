# /new-case-study

Kick off a new portfolio case study / interactive prototype.

Steps:
1. Ask for the project name and the one-line outcome it proves.
2. Create `docs/features/<slug>.md` capturing: the user (program lead / design leader),
   the thesis, what the interaction proves, scope, and all required states.
3. Add an entry to `docs/open-questions.md` and a row to `docs/changelog.md`.
4. Stub a flow in `reference/flows/<slug>/`.
5. Propose on-system patterns from `design-system/components.md`, using `var(--…)` tokens,
   Lucide icons, and a single accent per screen.
