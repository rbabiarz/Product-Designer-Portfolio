# Rule: Code style

- **Stack:** Self-contained HTML — single-file pages with inline CSS/JS, no build step and no server. `.dc.html` pages embed markup in an `<x-dc>` template that `support.js` renders with React (loaded from the unpkg CDN). Icons via Material Symbols from the MUI design kit (`<span class="msi" aria-hidden="true">icon_name</span>`).
- **Self-contained single files:** all CSS and JS inline; only declared CDN deps (React,
  fonts) at the top. No build step — edit the file, refresh the browser.
- **Icons:** Material Symbols (MUI kit): `<span class="msi" aria-hidden="true">icon_name</span>` — subsetted font link + `.msi` class live in each page's `<head>`; add new names to the `icon_names=` list when introducing an icon. Icon-only controls keep an `aria-label`. Never write custom SVG paths unless asked.
- **Navigation / screens:** JS `showSection()`-style toggles for multi-screen simulation;
  `.dc.html` markup lives inside the `<x-dc>` template.
- **Tokens over literals:** reference `var(--…)`; no raw hex/px in component code.
- **Shared runtime modules** (`support.js`, `a11y.js`, `text-motion.js`, `page-transition.js`,
  `home-variants.js`, `colophon.js`) stay at the repo root and must keep the site working with
  no server. `colophon.js` is generated by `build-colophon.py` — regenerate, never hand-edit.
- **Robust in embeds:** prototypes render inside host wrappers that may carry a `transform`;
  avoid relying on viewport-fixed positioning where an ancestor transform can break it
  (anchor overlays to their stage). Run isolated inits so one failure can't abort the rest.
