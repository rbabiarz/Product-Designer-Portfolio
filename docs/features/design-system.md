# The Design System — This Site, Documenting Itself

**User:** a design leader or technical hiring manager deciding whether "design engineering,
AI-paired, verification-gated" is a practiced discipline here or a slogan; the case is the
portfolio's capstone — the argument the other ten case studies make, applied to their own making.

**Thesis:** portfolios assert; this one runs. 29 self-contained pages on a three-tier token
system (232 values, semantic naming, a written never-hardcode rule), zero build steps, built with
Claude Code as a standing pair against a machine-readable brief and gated by Playwright
verification — with the numbers derived, never typed.

**What the pages prove:** the quick tour shows the system's three homepage faces, the token
discipline, the playable evidence layer, and the AI-paired loop. The deep dive adds two live
instruments (`ds-live.js`): a token inspector reading the computed custom properties of the page
it sits on, and a "run the receipts" panel that re-derives the Parlay slot's exact RTP in the
visitor's browser — reproducing both the bugged engine the adversarial review caught (95.9160%)
and the shipped fix (96.0428%) from the same constants the slot spins on. Trends section cites
primary sources (DTCG 2025.10, Figma's 2025 AI report, EAA enforcement, the buildless-web
movement). Honest limits stated: four pages carry open audit findings, two token systems is a
tradeoff, the mobile switcher menus are still hand-curated.
